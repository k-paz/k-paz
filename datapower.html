Linux/Debian     Proxmox    DataPower
Deploy IBM DATAPOWER by importing OVA/OVF from VMWARE on Proxmox8

###
#1 Obtain your DataPower VM in the OVA/OFA image - either by directly downloading the new, fresh image for clean installation from IBM Software Access pages or export your former VM image from Vmware/ESX/vSphere servers and out it on your Proxmox server somewhere:

cd /IBMv3700/DANE/DataPowerVMs/idg-v10-540/
wget https://download.source.of.your.file/idg10540.cd.dev.ova
scp idg10540.cd.dev.ova …

#2 unpack the ova image for further proxmox processing on your Proxmox server:

tar xvf idg10540.cd.dev.ova

#3 switch to root user on your Proxmox server:

su -

#4 as root list your current VMs to determine their IDs and assume on some unused IDs:

qm list

#5 as root list your storage options (names from 1st column of the listing) on Proxmox:

pvesm status

#6 as root import the OVF file that your previously unpacked from OVA (step 2) - with assigning some unused VM ID (eg. vmid=400) for your new DataPower VM on chosen Proxmox storage (eg. IBMv3700-PVEdata) name:

cd /IBMv3700/DANE/DataPowerVMs/idg-v10-540/
qm importovf 400 idg10540.cd.dev-signed.ovf IBMv3700-PVEdata

#7 as root update your new DataPower VM configuration to correct the CPU, Memory and other params (note that bridge=vmbr9 should be also adjusted to fit your network interfaces in Proxmox):

qm set 400 --sockets 1 --cores 4 --memory 16384 --cpu host --machine q35 --scsihw virtio-scsi-pci --boot order="scsi0;scsi1" --name NewDataPowerVM --acpi 1 --balloon 0 --net0 vmxnet3,bridge=vmbr9 --net1 vmxnet3,bridge=vmbr9 --net2 vmxnet3,bridge=vmbr9 --net3 vmxnet3,bridge=vmbr9

#8 Now your DataPower VM is ready to be used, however if your are preparing some educational env. for training or workshops, then you might also want to quickly duplicate your clean DPG VM for more users each with own VM:

qm clone 400 401 --full --name DP_VM_01
qm clone 400 402 --full --name DP_VM_02

#9 Done - ready to go. Just one more thing - if you have chosen bridge=vmbr9 for your internal/in-memory virtual network, then please remember that it might be useful to redirect some of your internal DataPower VM's IPs onto external port for direct admin access via browser for users:

redir --lport=30040 --caddr=192.168.9.40 --cport=9090 &
redir --lport=30041 --caddr=192.168.9.41 --cport=9090 &
redir --lport=30042 --caddr=192.168.9.42 --cport=9090 &
